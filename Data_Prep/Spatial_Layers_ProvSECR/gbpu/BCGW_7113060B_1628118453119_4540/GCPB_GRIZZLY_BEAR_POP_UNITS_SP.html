Grizzly Bear Population Units

1680 views (10 recent)

Published by the Ministry of Forests, Lands, Natural Resource Operations and Rural Development - Wildlife and Habitat
Licensed under Open Government Licence - British Columbia

Boundaries identifying similar behavioural ecotypes and sub-populations of Grizzly bears. This dataset contains versions from multiple years.

From 2018 on, NatureServe conservation concern ranking categories (e.g., Very Low, Low, Moderate, High, Extreme Concern) supersede the pre-2018 population status categories (e.g., Viable, Threatened, Extirpated) contained in the field STATUS. NatureServe conservation concern ranking categories reflect population size and trend, genetic and demographic isolation, as well as threats to bears and their habitats. The NatureServe conservation concern ranking fields are named CONSERVATION_CONCERN_RANK and CONSERVATION_CONCERN_DESC. Please view the attached PDF file for a summary of changes to this dataset from 2012 onward.

To download only the 2018 units, in the link below, select the "Export" tab, then select the "Provincial Layer Download" button: https://maps.gov.bc.ca/ess/hm/imap4m/?catalogLayers=7744,7745

Grizzly Bear Conservation Ranking results table is available here: https://catalogue.data.gov.bc.ca/dataset/e08876a1-3f9c-46bf-b69a-3d88de1da725

Grizzly Bear population estimates from various years are available here: https://catalogue.data.gov.bc.ca/dataset/2bf91935-9158-4f77-9c2c-4310480e6c29

Grizzly Bear reports are available here: https://www2.gov.bc.ca/gov/content/environment/plants-animals-ecosystems/wildlife/wildlife-conservation/grizzly-bear

Additional Information

Purpose

Grizzly Bear Population Units (GBPU) boundaries are meant to identify similar behavioural ecotypes and sub-populations of Grizzly bears.
Data Quality

This dataset contains multiple versions of the Grizzly Bear Population Units (GBPU) polygons as overlapping geometries, which can be separated by querying the VERSION_NAME field.
Lineage Statement

The Grizzly Bear Population Units (GBPU) polygon boundaries in this datasets go back to the year 2003. The previous GBPU boundary versions are from 2003a, 2003b, 2005 and 2012.

The 2015 boundary adjustments were directed by Tony Hamilton (Wildlife Biologist) and prepared by Sasha Lees (GIS).

The 2018 version has no boundary changes, but uses the Conservation Ranking attributes. The 2018 boundaries and population estimates were used in the 2018 provincial Cumulative Effects Assessment.

The 2020 version includes a boundary update for the Kitlope-Fiordland GBPU, directed by Garth Mowat (Wildlife & Habitat Branch) and prepared by Rajiv Lalla (GIS).

Object Description

Object Name: WHSE_WILDLIFE_INVENTORY.GCPB_GRIZZLY_BEAR_POP_UNITS_SP

Short Name: GBPU_BC
Comments: Grizzly Bear Population Units identify similar behavioural ecotypes and sub-populations of Grizzly bears. This dataset contains multiple versions as overlapping geometries.

Column Name Short Name Data Type Data Precision Comments
GRIZZLY_BEAR_POP_UNIT_ID GBPU_ID NUMBER 10 Is a system generated unique identification number.
GRIZZLY_BEAR_POPULATION_TAG GBPU_TAG NUMBER 10 Is a business key for the population unit. It consists of a positive non-zero integer which can be null, e.g., 11.
POPULATION_NAME POP_NAME VARCHAR2 50 Is the business name for the population unit. The population name generally describes a geographic area of the province, e.g., Blackwater-West Chilcotin.
DISPLAY_NAME DISP_NAME VARCHAR2 60 Is the formatted display name to allow for label stacking, e.g., Blackwater-\West Chilcotin.
STATUS STATUS VARCHAR2 20 Is the status or health of a population in a grizzly bear population unit., e.g., Viable, Threatened, Extirpated.
CONSERVATION_CONCERN_RANK CNSRV_RANK VARCHAR2 10 Is the NatureServe conservation management concern rank (M1-M5) that reflects the population size and trend, genetic and demographic isolation, as well as threats to bears and their habitats per Grizzly Bear Population Unit (GBPU). After 2018, these NatureServe ranking categories supersede the previous grizzly bear population unit status in the field STATUS. In general terms, categories M4 and M5 replace the previous Viable category, M1-M3 are analogous to the previous Threatened category, and M1 requires the most urgent conservation management focus.
CONSERVATION_CONCERN_DESC CNSRV_DESC VARCHAR2 20 Is the description of the NatureServe conservation concern rank (M1-M5), i.e., M1 (Extreme Concern), M2 (High Concern), M3 (Moderate Concern), M4 (Low Concern) and M5 (Very Low Concern).
WITHIN_BC_IND WITHIN_BC VARCHAR2 3 Is the flag if the population units is in BC, e.g., Y, N.
VERSION_NAME VER_NAME VARCHAR2 6 Is the name of the version of the population units, e.g., 2003A, 2003B, 2005, 2012, 2015, 2018.
VERSION_YEAR_MODIFIED VER_YR_MOD NUMBER 4 Is the year of the modification to match for example, prior hunting regulations of the year.
FEATURE_AREA_SQM AREA_SQM NUMBER 19 Is the system calculated area of a two-dimensional polygon in square meters
FEATURE_LENGTH_M FEAT_LEN NUMBER 19 Is the system calculated length or perimeter of a geometry in meters
GEOMETRY GEOMETRY SDO_GEOMETRY 1 Is the column used to reference the spatial coordinates defining the feature.
OBJECTID OBJECTID NUMBER 38 Is a required attribute of feature classes and object classes in a geodatabase.
SE_ANNO_CAD_DATA BLOB 4000 Is a binary column used by spatial tools to store annotation, curve features and CAD data when using the SDO_GEOMETRY storage data type.
